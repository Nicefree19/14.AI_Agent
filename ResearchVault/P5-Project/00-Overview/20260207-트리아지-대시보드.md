# 🎯 P5 이슈 트리아지 대시보드

## 🔥 Critical 우선순위

```dataview
TABLE WITHOUT ID
  link(file.link, title) as "제목",
  owner as "담당자",
  due_date as "마감일",
  category as "카테고리"
FROM "P5-Project/01-Issues"
WHERE issue_id AND priority = "critical"
SORT due_date ASC
```

## 🔴 High 우선순위

```dataview
TABLE WITHOUT ID
  link(file.link, title) as "제목",
  owner as "담당자",
  due_date as "마감일",
  issue_status as "상태"
FROM "P5-Project/01-Issues"
WHERE issue_id AND priority = "high" AND issue_status != "closed"
SORT due_date ASC
LIMIT 20
```

## 👤 담당자별 현황

```dataview
TABLE length(rows) as "건수"
FROM "P5-Project/01-Issues"
WHERE issue_id AND issue_status != "closed"
GROUP BY owner as "담당자"
SORT rows.length DESC
```

## 🏢 발신조직별 현황

```dataview
TABLE length(rows) as "건수"
FROM "P5-Project/01-Issues"
WHERE issue_id AND source_origin
GROUP BY source_origin as "발신조직"
```

## 🏷️ 키워드 히트 현황

> [!note] `keywords_hit` 필드는 `p5_email_triage.py`의 트리아지 결과 반영(BUG #6 수정) 후 채워집니다.

```dataview
TABLE length(rows) as "건수",
      rows.priority as "우선순위"
FROM "P5-Project/01-Issues"
WHERE issue_id AND keywords_hit
GROUP BY keywords_hit as "키워드"
```

## ⚠️ 예외 현황

### 담당자 미지정

```dataview
TABLE WITHOUT ID
  link(file.link, title) as "제목",
  priority as "우선순위",
  category as "카테고리"
FROM "P5-Project/01-Issues"
WHERE issue_id AND (!owner OR owner = "")
SORT priority DESC
LIMIT 10
```

### 마감일 없는 High/Critical

```dataview
TABLE WITHOUT ID
  link(file.link, title) as "제목",
  priority as "우선순위",
  owner as "담당자"
FROM "P5-Project/01-Issues"
WHERE issue_id AND (!due_date OR due_date = "") AND (priority = "high" OR priority = "critical")
SORT priority DESC
LIMIT 10
```
