# 연구 프로세스 워크플로우

## 단계별 프로세스

### 1. 주제 정의
- 연구 질문 명확화
- 키워드 목록 작성
- 범위(scope) 설정

### 2. 자료 수집
- [[source-collection]] 절차 따르기
- 수집한 자료마다 출처 노트 생성 ([[source-note]] 템플릿 사용)
- `#status/inbox` 태그 부여

### 3. 읽기 및 주석
- 자료 읽으며 핵심 내용 발췌
- 자신의 말로 재작성 (제텔카스텐 원칙)
- 영구 노트 생성 ([[research-note]] 템플릿 사용)

### 4. 연결 및 구조화
- [[note-linking]] 전략에 따라 노트 간 링크 생성
- MOC(Map of Content) 노트로 주제별 정리
- 그래프 뷰로 연결 구조 확인

### 5. 종합 및 분석
- [[synthesis]] 스킬 적용
- 패턴, 갈등, 빈 영역 파악
- 결론 도출 및 문서화

### 6. NotebookLM 동기화
- watchdog가 변경 감지 → NotebookLM에 자동 동기화
- NotebookLM에서 AI 기반 질의/분석 수행
- 결과를 다시 Obsidian에 기록

## 상태 전환
```
inbox → draft → review → final → archive
```
